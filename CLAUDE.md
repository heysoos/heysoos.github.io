# Portfolio Website

Personal portfolio for Sina Khajehabdollahi — computational neuroscientist and ALIFE researcher. The site is both a professional CV and a living gallery of WebGPU simulation art. Design philosophy: dark, minimal frame where simulations provide all the visual richness.

## Commands

```bash
npm run dev       # Astro dev server with hot reload
npm run build     # Static build → dist/
npm run preview   # Preview the built site locally
```

Deploys automatically to GitHub Pages on push to `main` via `.github/workflows/deploy.yml`.

## Stack

- **Framework:** Astro (static, no SSR)
- **Language:** TypeScript
- **GPU:** WebGPU with WGSL shaders
- **Styling:** CSS custom properties (theme system)
- **Content:** Astro content collections (Markdown)

## Architecture

```
src/
  components/simulations/   # WebGPU simulations, each self-contained
    boids/                  #   Boids.astro + boids-controller.ts + boids.wgsl
                            #   boids-grid.wgsl (spatial grid) + boids-panel.ts (UI panel)
                            #   trail-renderer.ts + trail.wgsl (trail subsystem)
    particle-life/          #   stub
    nca/                    #   stub
    cppn/                   #   stub
  lib/webgpu/               # Shared: device.ts, utils.ts, obstacle-tracker.ts, preview-registry.ts
  data/
    boids-presets.ts        # AUTO-GENERATED by /admin/boids — do not edit manually
    boids-shaders/          # Named WGSL preset shaders (snake.wgsl, chain-link.wgsl, etc.)
  content/
    publications/           # Markdown with frontmatter (title, authors, year, doi, featured)
    blog/                   # Markdown posts (title, date, tags)
    projects/               # Markdown gallery entries (slug, simulation, order)
  layouts/
    BaseLayout.astro
    SimLayout.astro         # Layout for full-viewport simulation pages
  pages/
    index.astro             # Hero boids + bio teaser + publications + gallery teaser
    about.astro
    contact.astro
    publications.astro
    blog/[...slug].astro
    gallery/index.astro
    gallery/[...slug].astro # Full-viewport simulation + controls
    admin/boids.astro       # Dev-only preset editor (non-functional in production)
  styles/themes/            # warm-ember.css (default), deep-space, monochrome, muted-violet
  config.ts                 # Site-wide config: name, tagline, social links, nav, theme
```

## Simulations

Each simulation lives in `src/components/simulations/<name>/` with core files:
- `<name>.astro` — component wrapper
- `<name>-controller.ts` — WebGPU setup, buffers, render loop
- `<name>.wgsl` — compute + render shaders
- Additional files (e.g. `trail-renderer.ts`, `trail.wgsl`) for subsystems like trails

Pipeline: check WebGPU support → request adapter/device → create buffers/pipelines → rAF loop (compute pass → render pass).

WebGPU unavailability: handle inline in each controller — no shared fallback file.

## Adding a New Simulation

1. Create `src/components/simulations/<name>/` with the three files above
2. Add a Markdown file to `src/content/projects/` with `simulation: "<name>"`
3. It automatically appears in the gallery

## Theme System

Active theme is set by a single import in `BaseLayout.astro`. To switch: change the import to a different file in `src/styles/themes/`. All colors are CSS custom properties — the key tokens are `--bg-primary`, `--bg-surface`, `--text-primary`, `--text-body`, `--accent`, `--accent-glow`.

## Gotchas

- WebGPU requires a secure context (HTTPS or localhost). The dev server runs on HTTP localhost, which is fine.
- Dynamic imports for WebGPU components use `client:only="react"` is NOT the pattern here — Astro island with `client:only` or inline `<script>` tags in `.astro` files.
- Gallery page uses dynamic imports with error handling to avoid SSR crashes (WebGPU is browser-only).
- `astro.config.mjs` has GitHub Pages base path configured — don't break this.
- Boids has a live in-browser shader editor (CodeMirror) — `reloadShader()` in the controller hot-swaps WGSL at runtime.
- `src/data/boids-presets.ts` is AUTO-GENERATED — edit presets via `/admin/boids` during dev, not by hand.
- The admin preset-save API (`/api/admin/save-presets`) is a Vite plugin in `astro.config.mjs`. It only exists during `npm run dev`; the page is silently non-functional in production.
- There is no shared `fallback.ts` — WebGPU unavailability is handled inline in each controller.
