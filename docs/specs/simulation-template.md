# Simulation Implementation Template

Conventions for adding WebGPU simulations to the gallery. Refer to `boids` and `cppn` as canonical examples.

## File Structure

| File | Location | Purpose |
|------|----------|---------|
| `<name>-controller.ts` | `src/components/simulations/<name>/` | WebGPU setup, render loop, public params |
| `<name>-panel.ts` | `src/components/simulations/<name>/` | UI panel builder |
| `<name>-presets.ts` | `src/data/` | Auto-generated preset array — do not edit |
| `admin/<name>.astro` | `src/pages/` | Dev-only preset editor |
| `<name>.md` | `src/content/projects/` | Gallery metadata |

## Naming Conventions

| Concept | Pattern | Example |
|---------|---------|---------|
| Controller class | `<Name>Controller` | `BoidsController` |
| Panel function | `build<Name>Panel` | `buildBoidsPanel` |
| Preset interface | `<Name>Preset` | `BoidsPreset` |
| Presets constant | `<NAME>_PRESETS` | `BOIDS_PRESETS` |

## Controller

Required interface:

```ts
class <Name>Controller {
  async init(canvas: HTMLCanvasElement): Promise<boolean>
  start(): void
  stop(): void
  reset(): void
}
```

- Use `initWebGPU(canvas)` from `src/lib/webgpu/device.ts`
- Runtime params are public properties mutated directly by the UI — no event system
- `init` returns `false` on WebGPU unavailability; handle inline, no shared fallback

## Panel Builder

```ts
export function build<Name>Panel(
  container: HTMLElement,
  controller: <Name>Controller,
  opts: { presets?, activePresetId?, onPresetLoad?, onClose? }
): void
```

Generates DOM manually. Reads/writes controller properties directly via getter/setter closures.

## Presets

Minimum shape:

```ts
interface <Name>Preset {
  id: string
  name: string
  isDefault?: boolean
  // ...simulation-specific fields
}
```

The presets file is auto-generated by the admin page. The save endpoint is a Vite dev middleware registered in `astro.config.mjs` at `/api/admin/save-<name>-presets`.

Heavy preset data (shader source, weight arrays, etc.) should not be inlined in the `.ts` file. Instead, store them as separate assets (e.g. `.wgsl` files imported with `?raw`, or binary/JSON files) and reference them from the preset object. The code generator is responsible for producing the correct imports.

## Gallery Wiring (`src/pages/gallery/[...slug].astro`)

1. Import the controller and add it to the `controllers` record keyed by `"<name>"`
2. Add a conditional block for sim-specific init: load default preset, call `build<Name>Panel`

## Preview Registry (`src/lib/webgpu/preview-registry.ts`)

Optional. Add a case that dynamically imports the controller, inits with reduced settings, and returns `{ pause, resume }`.

## Content Entry (`src/content/projects/<name>.md`)

```yaml
---
title: "..."
description: "..."
simulation: "<name>"
order: N
---
```

The `simulation` field must match the key used in the `controllers` record.
