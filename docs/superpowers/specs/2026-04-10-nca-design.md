# NCA Simulator Design

**Date:** 2026-04-10
**Variant:** Mordvintsev-style Growing NCA (minimalNCA)
**Status:** Approved

---

## Overview

A WebGPU Neural Cellular Automata simulator for the gallery. Implements the Mordvintsev growing NCA: fixed depth-wise perception filters feeding a small per-cell MLP with a stochastic update mask. Supports trained weight presets (extracted from `.pth` files) and random-init exploration with live architecture configuration and recompile.

---

## Core Algorithm

Per-step update (runs `stepsPerFrame` times per animation frame):

1. **Perception** — apply toggled fixed filters depth-wise to each channel:
   - Identity `[[0,0,0],[0,1,0],[0,0,0]]`
   - Sobel X `[[-1,0,1],[-2,0,2],[-1,0,1]] / 8`
   - Sobel Y (Sobel X transposed) `/ 8`
   - Laplacian `[[1,2,1],[2,-12,2],[1,2,1]] / 16`
   - Produces `CHANNELS × N_FILTERS` values per cell (N_FILTERS = number of active filters, max 4)

2. **MLP** — per cell:
   - `Linear(CHANNELS × N_FILTERS → HIDDEN)` + activation
   - `Linear(HIDDEN → CHANNELS)` → delta (no bias on second layer)

3. **Stochastic mask** — each cell independently fires with probability `fireRate`

4. **Residual update** — `state += delta`, clamped to `[0, 1]`

Grid wrapping is toroidal (circular padding in all directions).

## GPU Pipeline

### Buffers

- **State buffers A & B** — flat `f32` storage buffers of size `[gridWidth × gridHeight × CHANNELS]`, ping-ponged each step
- **Weight buffer** — flat `f32` storage buffer, layout: `[W1_weights | W1_biases | W2_weights]`
  - W1 weights: `CHANNELS × N_FILTERS × HIDDEN` floats
  - W1 biases: `HIDDEN` floats
  - W2 weights: `HIDDEN × CHANNELS` floats (no bias)
- **Uniform buffer** — `fireRate`, `gridWidth`, `gridHeight`, `channelR/G/B`, `dt`, `frameIndex` (u32, incremented each step, used for per-cell stochastic RNG via `pcg_hash(cellIndex ^ frameIndex)`)

### Passes per frame

1. **Compute pass** (`nca-update.wgsl`, generated) — one workgroup invocation per cell, reads state A, writes state B, swaps references
2. **Render pass** (`nca-render.wgsl`, static) — fullscreen quad, reads state buffer, maps `channelR/G/B` → RGB display

### Codegen (`nca-codegen.ts`)

Generates WGSL with architecture baked in as constants:
```wgsl
const CHANNELS: u32 = 16u;
const HIDDEN: u32 = 64u;
const N_FILTERS: u32 = 4u;
// activation inlined as a function
```

Loop bounds are static — compiler can unroll. Weights come from the storage buffer — no recompile needed for weight changes.

Recompile is triggered only by: channels, hidden, filter set, or activation function changes.

## Types (`nca-types.ts`)

```ts
type NCAActivation = 'relu' | 'tanh' | 'leakyrelu';

interface NCAFilters {
  identity: boolean;
  sobelX: boolean;
  sobelY: boolean;
  laplacian: boolean;
}

interface NCAConfig {
  channels: number;           // 8 | 16 | 32
  hidden: number;             // 32 | 64 | 128
  filters: NCAFilters;
  activation: NCAActivation;
  fireRate: number;           // 0–1
  stepsPerFrame: number;      // 1–16
  dt: number;                 // step size multiplier, default 1.0
  gridWidth: number;          // 128 | 256 | 512
  gridHeight: number;
  channelR: number;           // display channel index
  channelG: number;
  channelB: number;
  normalizeDisplay: boolean;
}

interface NCAPreset {
  id: string;
  name: string;
  isDefault?: boolean;
  config: NCAConfig;
  weights: number[];          // flat Float32Array: [W1_weights | W1_biases | W2_weights]
}
```

## Controller API (`nca-controller.ts`)

```ts
class NCAController {
  async init(canvas: HTMLCanvasElement): Promise<boolean>
  start(): void
  stop(): void
  reset(): void                              // re-seeds grid per current seed mode
  recompile(config: NCAConfig): Promise<void> // regenerates WGSL, rebuilds pipeline
  loadPreset(preset: NCAPreset): void        // writeBuffer weights + apply config (recompile if architecture differs)
  randomInit(): void                         // new random weights CPU-side → writeBuffer
  setParams(partial: Partial<NCAConfig>): void
  brush(x: number, y: number, opts: BrushOptions): void
}

interface BrushOptions {
  mode: 'damage' | 'paint';
  shape: 'circle' | 'square';
  size: number;     // radius in pixels
  strength: number; // 0–1
}
```

## Brush / Interaction

Mouse down + drag on canvas → compute affected cell coordinates → `device.queue.writeBuffer` patch directly into the active state buffer. No compute pass needed.

- **Damage**: writes zeros to all channels in the patch
- **Paint**: writes random noise (values ~0.5–1.0) into all channels

Brush works mid-simulation without pausing.

Dev-only, non-functional in production (same pattern as `/admin/boids`, `/admin/cppn`).

**Random preset editor**: full panel + live preview + "Save as preset" → name input → POST to `/api/admin/save-nca` Vite middleware → writes `src/data/nca-weights/<id>.json` + regenerates `src/data/nca-presets.ts`.

**Trained preset importer**: upload a pre-extracted `.json` weight file, specify config, preview live, save.

Vite middleware endpoint `/api/admin/save-nca` added to `astro.config.mjs`.

## Weight Format

Flat `number[]` in layout order: `[W1_weights... | W1_biases... | W2_weights...]`

- W1 weights: row-major `[CHANNELS × N_FILTERS, HIDDEN]` — outer index is input neuron
- W1 biases: `[HIDDEN]`
- W2 weights: row-major `[HIDDEN, CHANNELS]`

This layout matches `writeBuffer` directly — no runtime reshaping needed.
